Do not skip this part

It is powerful because it is local. It is risky because it is local.

This is not a sandbox and not a hosted assistant. It gives ChatGPT a local control surface. That is exactly why it is useful — and exactly why supervision matters.

Start narrow. Use workspace boundaries and conservative modes first.

Backup before edits. Make recoverability part of the workflow.

Approve destructive work. Local power is not autonomous permission.

Trust evidence. Inspect logs, output and changed files.

For the impatient

Clone it. Run it. Connect it. Then start with inspection.
